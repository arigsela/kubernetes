# homelab-agent BYO Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy the `homelab-agent` container (image `…/homelab-agent:v0.2.0`) as a kagent BYO `Agent` CR with conversation memory enabled — a dedicated `homelab_agent` database in the in-cluster pgvector Postgres, scoped ESO/Vault credentials per the agent-identity contract, coexisting with `homelab-knowledge`.

**Architecture:** GitOps manifests under `base-apps/` (auto-synced by Argo CD). A Sync-hook Job provisions the memory DB/role; ESO chains materialize three secrets from scoped Vault paths across the `postgresql` and `kagent` namespaces; a BYO Agent CR wires those secrets into the container's env. Vault paths/policies/roles are provisioned once by the operator (commands provided). No NetworkPolicy (none exists; egress is open).

**Tech Stack:** Kubernetes YAML, Argo CD (directory-recurse Applications), External Secrets Operator (Vault provider, KV v2), Kyverno (auto-injects ECR pull secret + enforces agent-identity), pgvector Postgres, kagent 0.9.11 BYO Agent CRD.

## Global Constraints

Every task's requirements implicitly include all of these:

- **Branch:** work on `feat/homelab-agent-deployment` in `/Users/arisela/git/kubernetes`. Never `git add -A` / `git add .`; always `git add <explicit path>`.
- **No secrets in git or logs.** Manifests reference Vault via ESO; real credential values live only in Vault. The DB password is generated by the operator via `openssl rand -hex 32` (URL-safe, no DSN escaping needed) and never printed.
- **Scoped credentials (agent-identity invariant 1, enforced by CI + Kyverno):** every `ExternalSecret` resolves through a dedicated `SecretStore` (its own `eso-*` ServiceAccount + Vault role reading only its one path) — never the broad `vault-backend` store, never `remoteRef.key: kagent`.
- **No chart bump; no NetworkPolicy; no `imagePullSecrets`** (Kyverno `inject-ecr-pull-secret` auto-adds `ecr-registry` for `.dkr.ecr.` images in `kagent`).
- **Image is `…/homelab-agent:v0.2.0`** (pinned tag, not `:latest`).
- **Coexistence:** do not modify or remove `agents/homelab-knowledge.yaml`. Retirement is a later, separate step.
- **Validation, not unit tests** (these are manifests): each task validates with (a) YAML parses (`python3 -c "import yaml,glob; [list(yaml.safe_load_all(open(f))) for f in [...]]"`), (b) `python3 scripts/validate-agent-identity.py` exits 0, (c) cross-reference consistency (every `secretKeyRef`/`remoteRef` name+key matches its producer). Server dry-run/`kubectl` is NOT assumed available (no cluster access); if it is, add `kubectl apply --dry-run=client -f <file>` for core kinds.
- **Commit style:** conventional commits scoped to `homelab-agent`.

- [ ] **Step 4: Init Job (Sync hook)** — create `base-apps/postgresql/init-homelab-agent-db.yaml`. Mirrors `init-kagent-db.yaml`; role name `homelab_agent`, db `homelab_agent`, password from `homelab-agent-db-credentials`:

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: init-homelab-agent-db
  namespace: postgresql
  annotations:
    # Sync hook (idempotent, self-cleaning) — same rationale as init-kagent-db.
    argocd.argoproj.io/hook: Sync
    argocd.argoproj.io/hook-delete-policy: HookSucceeded
spec:
  ttlSecondsAfterFinished: 300
  backoffLimit: 5
  template:
    metadata:
      labels:
        app: init-homelab-agent-db
    spec:
      nodeSelector:
        node.kubernetes.io/workload: application
      restartPolicy: OnFailure
      containers:
      - name: init-homelab-agent-db
        image: pgvector/pgvector:0.8.2-pg18
        command:
        - /bin/sh
        - -c
        - |
          set -e
          PG_HOST="postgresql.postgresql.svc.cluster.local"
          PG_PORT="5432"
          ESCAPED_PASSWORD=$(echo "$HOMELAB_PASSWORD" | sed "s/'/''/g")
          until pg_isready -h "$PG_HOST" -p "$PG_PORT" -U "$POSTGRES_USER"; do
            echo "Waiting for PostgreSQL..."; sleep 2
          done
          PGPASSWORD="$POSTGRES_PASSWORD" psql -h "$PG_HOST" -p "$PG_PORT" -U "$POSTGRES_USER" -d "$POSTGRES_DB" \
            -v homelab_user="homelab_agent" \
            -v homelab_password="$ESCAPED_PASSWORD" \
            -v homelab_db="homelab_agent" <<'EOSQL'
            SELECT format('CREATE ROLE %I WITH LOGIN PASSWORD %L', :'homelab_user', :'homelab_password')
            WHERE NOT EXISTS (SELECT FROM pg_catalog.pg_roles WHERE rolname = :'homelab_user')\gexec
            SELECT format('ALTER ROLE %I WITH PASSWORD %L', :'homelab_user', :'homelab_password')
            WHERE EXISTS (SELECT FROM pg_catalog.pg_roles WHERE rolname = :'homelab_user')\gexec
            SELECT format('CREATE DATABASE %I OWNER %I', :'homelab_db', :'homelab_user')
            WHERE NOT EXISTS (SELECT FROM pg_database WHERE datname = :'homelab_db')\gexec
          EOSQL
          PGPASSWORD="$POSTGRES_PASSWORD" psql -h "$PG_HOST" -p "$PG_PORT" -U "$POSTGRES_USER" -d "homelab_agent" <<'EOSQL'
            CREATE EXTENSION IF NOT EXISTS vector;
          EOSQL
          echo "homelab_agent database initialized successfully with pgvector extension"
        env:
        - name: POSTGRES_USER
          valueFrom: { secretKeyRef: { name: postgresql-credentials, key: n8n-user } }
        - name: POSTGRES_PASSWORD
          valueFrom: { secretKeyRef: { name: postgresql-credentials, key: n8n-password } }
        - name: POSTGRES_DB
          valueFrom: { secretKeyRef: { name: postgresql-credentials, key: database-name } }
        - name: HOMELAB_PASSWORD
          valueFrom: { secretKeyRef: { name: homelab-agent-db-credentials, key: password } }
        resources:
          requests: { memory: "64Mi", cpu: "50m" }
          limits: { memory: "128Mi", cpu: "100m" }
```

````markdown
# homelab-agent — Vault provisioning & rollout runbook

One-time operator actions (run with a Vault root/admin token) to back the
ESO manifests in `base-apps/postgresql/` and `base-apps/kagent/`. Nothing here
is committed with real values; the DB password is generated inline.

## 1. Vault secrets

```bash
# DB password — URL-safe hex so the DSN needs no escaping. Never printed.
vault kv put k8s-secrets/homelab-agent-db password="$(openssl rand -hex 32)"

# App tokens — replace the placeholders with the real values.
#  - anthropic-api-key: a dedicated Anthropic key for this agent (NOT the shared kagent one)
#  - backstage-token:   the Backstage MCP bearer token value (same one backstage-catalog-mcp uses)
vault kv put k8s-secrets/homelab-agent \
  anthropic-api-key='REPLACE_WITH_ANTHROPIC_KEY' \
  backstage-token='REPLACE_WITH_BACKSTAGE_MCP_TOKEN'
```

## 2. Vault policies (least privilege — each reads only its one path)

```bash
vault policy write homelab-agent - <<'EOF'
path "k8s-secrets/data/homelab-agent" { capabilities = ["read"] }
EOF

vault policy write homelab-agent-db - <<'EOF'
path "k8s-secrets/data/homelab-agent-db" { capabilities = ["read"] }
EOF
```

## 3. Kubernetes-auth roles

```bash
# App-secrets: eso-homelab-agent in kagent only.
vault write auth/kubernetes/role/homelab-agent \
  bound_service_account_names=eso-homelab-agent \
  bound_service_account_namespaces=kagent \
  policies=homelab-agent ttl=1h

# DB password/DSN: eso-homelab-agent-db in BOTH kagent (agent DSN) and
# postgresql (init job). One role, two namespace bounds.
vault write auth/kubernetes/role/homelab-agent-db \
  bound_service_account_names=eso-homelab-agent-db \
  bound_service_account_namespaces=kagent,postgresql \
  policies=homelab-agent-db ttl=1h
```

## 4. Roll out (merge the manifests, let Argo sync) and verify

```bash
# ESO materialized the three secrets:
kubectl -n postgresql get externalsecret homelab-agent-db-credentials   # STATUS SecretSynced
kubectl -n kagent     get externalsecret homelab-agent-secrets          # SecretSynced
kubectl -n kagent     get externalsecret homelab-agent-db               # SecretSynced

# DB provisioned (idempotent hook):
kubectl -n postgresql logs job/init-homelab-agent-db   # "...initialized successfully with pgvector extension"

# Agent up, serving A2A + memory:
kubectl -n kagent get pods -l app.kubernetes.io/name=homelab-agent
kubectl -n kagent port-forward svc/homelab-agent 18080:8080 &
curl -s localhost:18080/health                          # {"status":"healthy","agent":"homelab-agent"}
curl -s localhost:18080/.well-known/agent.json | jq '.skills[].id'   # the three skill ids
```

## 5. Parity & cutover (later, separate)

Compare `homelab-agent` vs `homelab-knowledge` over A2A on the golden
questions (the skill examples). Once parity holds, retire the Declarative
`homelab-knowledge` CR in a follow-up PR. Reversible at each step.

## Rotation

`vault kv put k8s-secrets/homelab-agent-db password=...` then re-run the init
job (ALTER ROLE is idempotent) and restart the agent pod so ESO re-renders the
DSN. Same for `homelab-agent` app tokens (ESO refresh is hourly, or delete the
target Secret to force an immediate resync).
````

## Open items resolved during planning (for the record)

- **ECR pull:** Kyverno `inject-ecr-pull-secret` auto-adds `imagePullSecrets: [{name: ecr-registry}]` for `.dkr.ecr.` images in `kagent`; `ecr-registry` is auto-cloned into the namespace. Do NOT set `imagePullSecrets`. ✓
- **agent-identity gates vs BYO:** both the Kyverno `ClusterPolicy` and `scripts/validate-agent-identity.py` match the Agent by kind but their model/toolNames checks read `spec.declarative.*`, which a BYO agent lacks — so they no-op (pass) on it. The ExternalSecrets ARE fully gated and are scoped correctly. ✓
- **admin login for the init job:** `postgresql-credentials` keys `n8n-user`/`n8n-password`/`database-name` (same as `init-kagent-db`). ✓
- **dual-namespace DB role:** one Vault role `homelab-agent-db` with `bound_service_account_namespaces=kagent,postgresql`. ✓
